Полный MVP: лендинг + интерактивные инструменты + личный кабинет + PRO-подписка.

## 🎨 Дизайн-система

**Тёмная премиум-тема (киберспорт + про-инструмент):**
- Фон: глубокий тёмно-синий/чёрный (`#0B0F19`), карточки чуть светлее
- Акцент: тёплый янтарный/золотой (`#F5A524`) — для CTA, PRO, активных состояний
- Семантика: зелёный для favorable winrate, красный для unfavorable, нейтрально-серый для ~50%
- Шрифты: Inter для UI, Space Grotesk для заголовков
- Плавные переходы, лёгкие свечения на hover, скруглённые углы, матовые карточки с тонкой рамкой
- Полная адаптивность (mobile + desktop)

## 📊 3. Matchups (`/matchups`)

Интерактивная таблица матчапов архетипов колод Hearthstone (Aggro Paladin, Control Warrior, Big Priest, Pirate DH и т.д.):

- Строки/столбцы = архетипы (с иконкой класса)

- **Tooltip** в ячейке: число игр, тренд, последнее обновление
- Сортировка по среднему winrate
- Селектор **меты/патча** (например, "Patch 28.6", "Showdown in the Badlands")
- Переключатель **формата** (Standard / Wild / Twist)
- Sticky первая колонка/строка для удобного чтения

## 🛡️ 4. Ban Tool (`/ban-tool`) — главная фича

Conquest-формат (стандарт для турниров Hearthstone):

**Ввод:**
- Выбор формата лайнапа: 2 / 3 / 4 deck Conquest
- Поля для **наших колод** (с автокомплитом по архетипам)
- Поля для **колод соперника** (с автокомплитом)
- Drag & Drop для перестановки

**Расчёт** (на основе winrate-матрицы):
- Перебираем все варианты бана колоды соперника
- Для каждого считаем матожидание победы по Conquest (нужно выиграть всеми колодами по одному разу)
- Возвращаем **лучший бан** с вероятностью победы + топ-3 альтернатив

**Вывод:**
- Большая карточка с рекомендуемым баном (иконка колоды + % win probability)
- Прогресс-бар вероятности
- Список альтернатив с их % (отсортирован)
- Мини-визуализация: bar chart сравнения

UI: кнопка «Рассчитать», состояние loading, плавная анимация результата.

## 👤 7. Авторизация и Profile (`/profile`)

**Auth (Lovable Cloud):**
- Email + password
- Google sign-in
- Защищённые роуты для `/profile`

**Личный кабинет:**
- Вкладки: История расчётов · Сохранённые стратегии · Подписка · Настройки
- **История:** последние расчёты Ban Tool (формат, лайнапы, рекомендованный бан, дата) — кнопка «Повторить»
- **Стратегии (PRO):** именованные сохранённые лайнапы для турниров
- **Подписка:** текущий статус (Free / PRO), дата окончания, кнопка управления / купить PRO
- **Настройки:** имя, аватар, смена пароля, удаление аккаунта

## 💳 8. PRO-подписка (Stripe)

Встроенная Stripe-интеграция Lovable (без своего аккаунта Stripe для старта):
- Один продукт «PRO» с месячной и годовой ценой
- Checkout session при клике «Купить PRO»
- Webhook обновляет статус подписки в БД
- Гейтинг функций: сохранение стратегий, расширенная статистика матчапов, кастомные турниры, приоритет в обновлениях
- Бейдж «PRO» рядом с именем в профиле

*Примечание: после утверждения плана я отдельно запущу проверку и подключу Stripe-платежи через built-in интеграцию.*

## 🚀 9. Onboarding для новых пользователей

Лёгкий 3-шаговый тур после регистрации (overlay-подсказки):
1. «Здесь смотришь матчапы» → Matchups
2. «Здесь рассчитываешь баны» → Ban Tool
3. «Сохраняй стратегии в профиле» → Profile

Можно пропустить, состояние «прошёл онбординг» хранится в профиле.

## 🗄️ Структура данных (Lovable Cloud)

- `profiles` — id, display_name, avatar_url, onboarding_completed
- `user_roles` — отдельная таблица ролей (admin / user) с `has_role()` функцией
- `subscriptions` — user_id, plan (free/pro), status, current_period_end, stripe_customer_id
- `archetypes` — id, name, class, icon, format
- `matchups` — archetype_a, archetype_b, winrate, games_count, patch
- `patches` — id, name, released_at, format
- `news` — id, title, slug, body, author, category, cover_url, published_at
- `calculations_history` — user_id, format, our_decks, opp_decks, recommended_ban, win_probability, created_at
- `saved_strategies` (PRO) — user_id, name, format, our_decks, opp_decks, notes
- `contact_messages` — email, message, created_at

Все таблицы с RLS: пользователь видит только свои записи; матчапы/новости/архетипы — публичное чтение.
